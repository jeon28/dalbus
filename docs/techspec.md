# [기술 제안서] 달버스(Dalbus) 웹 서비스 구현 가이드

본 문서는 비개발자 운영자가 서비스를 클라우드 환경에서 모바일과 PC 모두 대응(반응형)할 수 있도록 최적의 기술 스택과 구현 전략을 정리한 문서입니다.

---

## 1. 추천 기술 스택 (Tech Stack)

나중에 개발자를 채용하거나 외주를 맡길 때 가장 대중적이고 유지보수가 쉬운 **'자바스크립트/타입스크립트'** 생태계를 추천합니다.

| 구분 | 추천 기술 | 비고 |
| :--- | :--- | :--- |
| **프론트엔드** | **Next.js (React)** | PC/모바일/패드 모두 대응하는 **반응형 웹** 구현에 최적 |
| **백엔드** | **Node.js (NestJS)** | 가볍고 빠르며, 추후 '자동 매칭' 기능 확장 시 유리함 |
| **데이터베이스** | **PostgreSQL (Supabase)** | 비개발자도 엑셀처럼 데이터를 보고 수정하기 매우 편리함 |
| **클라우드 호스팅** | **Vercel** 또는 **AWS** | 배포가 간편하고 초기 사용량이 적을 때 비용이 거의 없음 |
| **디자인 툴** | **Figma** | 개발 전 화면 설계 및 기획안 작성용 |

---

## 2. 주요 외부 연동 서비스 (API)

직접 개발하지 않고 이미 만들어진 서비스를 연결하여 시간과 비용을 절약합니다.

1.  **본인인증 및 결제**: **포트원 (PortOne)**
    * 휴대폰 본인 확인(가입 시) 및 카드/간편결제 기능을 한 번에 해결.
2.  **자동 알림**: **알림톡 API (솔라피 등)**
    * 관리자가 '작업 완료' 클릭 시 사용자에게 카카오톡으로 계정 정보를 자동 전송.
3.  **관리자 대시보드 도구**: **Retool (리툴)**
    * 관리자가 ID/PW를 입력하고 주문을 관리하는 페이지를 코딩 없이 빠르게 구축 가능.

---

## 3. 단계별 개발 전략

### Step 1: 화면 설계 (와이어프레임)
* **피그마(Figma)**를 활용해 사용자가 보는 화면과 관리자가 보는 화면을 그립니다. 
* PC와 모바일에서 각각 어떻게 보일지 레이아웃을 정의합니다.

### Step 2: MVP(최소 기능 제품) 개발
* **회원가입/로그인**: 본인인증 포함.
* **Tidal 신청 페이지**: 결제 연동 및 이용약관 동의.
* **관리자 전용 페이지**: 입금자 명단 확인 및 ID/PW 입력창 구현. (가장 중요)
* **마이페이지**: 사용자가 발급된 정보를 확인하는 공간.

### Step 3: 클라우드 배포 및 운영
* **Vercel**을 통해 웹사이트를 실시간으로 띄웁니다.
* 사용자가 늘어남에 따라 서버 사양을 클릭만으로 확장할 수 있도록 구성합니다.

---

## 4. 데이터베이스 구조 (데이터 관리 핵심)

관리자가 시스템에 입력해야 할 데이터 항목입니다. (개발자 요청용)

* **Users(회원)**: 이름, 이메일(가입용), 휴대폰번호, 가입일, 본인인증값(CI/DI).
* **Orders(주문)**: 신청 서비스(Tidal), 이용기간, 결제상태(입금완료/대기), 작업상태(접수/작업중/완료).
* **Accounts(계정 정보)**: 배정된 Tidal ID, 배정된 비밀번호, 시작일, 종료일.

---

## 5. 비개발자 사장님을 위한 핵심 요청 사항

개발자 또는 외주 업체와 계약 시 반드시 아래 조건을 포함하세요.

1.  **반응형 웹 필수**: "PC용, 모바일용 따로 만들지 말고 하나로 다 되게 해주세요."
2.  **어드민 편의성**: "제가 직접 ID/PW를 입력하기 편하도록 관리자 페이지를 깔끔하게 만들어주세요."
3.  **확장성 고려**: "나중에 유튜브나 넷플릭스 메뉴를 제가 직접 추가할 수 있게 설계해 주세요."
